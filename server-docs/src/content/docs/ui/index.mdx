The Hytale UI system uses a declarative DSL (Domain-Specific Language) for defining user interfaces. UI files (`.ui`) describe layouts, styling, and interactive elements that can be displayed to players.

## Architecture

UI components are organized in `Assets/Common/UI/Custom/`:

```
Custom/
├── Common.ui          # Shared variables and base components
├── Sounds.ui          # Sound effect definitions
├── Common/            # Reusable component templates
│   ├── TextButton.ui
│   └── ActionButton.ui
├── Pages/             # Full-screen page layouts
│   ├── ShopPage.ui
│   ├── RespawnPage.ui
│   └── ...
└── Hud/               # In-game HUD elements
    └── TimeLeft.ui
```

## Basic Example

A simple page using the container system:

```ui
$C = "../Common.ui";

$C.@PageOverlay {}

$C.@Container {
  Anchor: (Width: 600, Height: 400);

  #Title {
    $C.@Title {
      @Text = "My Page";
    }
  }

  #Content {
    LayoutMode: Top;

    Label {
      Text: "Hello, World!";
      Style: $C.@DefaultLabelStyle;
    }

    $C.@TextButton {
      @Text = "Click Me";
    }
  }
}

$C.@BackButton {}
```

## Key Concepts

| Concept | Description |
|---------|-------------|
| `$Variable` | File import reference |
| `@Variable` | Local/component variable |
| `#ElementId` | Element identifier for scripting |
| `%localization.key` | Localized text reference |
| `...@Spread` | Property spread operator |
